# Quintet 分值设计说明

本文档记录 Phase 2 分值 Monte Carlo 模拟结果，以及 **v2 校准计分** 的设计 rationale。规则来源：[`prompt.md`](../prompt.md)。

## 1. 模拟方法

- **样本量：** 50,000 局
- **填充方式：** 从洗好的 52 张牌取 25 张，随机起点 + 随机合法邻接放置（与真实对局中“有合法位可放”一致）
- **计分线路：** 5 行 + 5 列 + 2 对角 = 12 路
- **牌型规则：** 标准德州扑克 5 张牌型；A 可作 1 组成 Wheel（A-2-3-4-5）
- **运行命令：**

```bash
cd prototype
python -m quintet.simulate --compare -n 50000
```

## 2. 牌型出现频率（每线路）

模拟中 v1 / v2 频率相同（仅分值不同）：

| 牌型 | 频率 |
|------|------|
| 高牌 | 50.16% |
| 一对 | 42.24% |
| 两对 | 4.74% |
| 三条 | 2.10% |
| 顺子 | 0.40% |
| 同花 | 0.19% |
| 葫芦 | 0.14% |
| 四条 | 0.03% |
| 同花顺 | ~0% |
| 皇家同花顺 | ~0% |

**解读：** 25 张牌占半副牌，每线路 5 张来自这 25 张的子集，因此「一对 / 高牌」占主导（合计 ~92%），强牌型罕见但可追求。12 线路重叠进一步使「多线同强」极难，符合策略深度预期。

## 3. v1 草案 vs v2 校准对比

| 指标 | v1 草案 | v2 校准 |
|------|---------|---------|
| 单局总分均值 | 152.38 | **123.54** |
| 单局总分标准差 | 19.51 | 16.90 |
| 95 分位 | 187.50 | 154.33 |
| 最高分 | 261.00 | 240.99 |
| 单线路均值 | 12.70 | **10.30** |

**v1 问题：**

- 总分偏高，高牌线路仍有 0~14 分，叠加 12 路后 baseline 过高
- 一对仅 `2 + 对子牌点`，与两对 / 三条梯度偏陡，中间档位激励不足
- 同花 `14 + 总和×0.2` 在强牌中偏低，与葫芦 / 四条倒挂风险

**v2 设计目标：**

- 单线路均值 ≈ **11** → 满盘约 **132** 分
- 各档基础分与出现频率大致反比
- 牌点微调（kickers）用 **小系数**，用于 tie-break，不改变档位感知

## 4. v4 计分公式（当前默认）

在 v3 基础上**压低常见牌型 baseline、抬高强牌档位**，使成顺/同花等 premium 对总局分的贡献更显著。20k 局模拟：单线路均值 ≈ **9.7**，满盘 ≈ **117** 分；含 ≥1 条顺子时均值 **~168**（v3 为 ~162）。

| 牌型 | 公式 |
|------|------|
| 皇家同花顺 | **200** |
| 同花顺 | **165** + 最大牌点 × 0.1 |
| 四条 | **130** + 四条牌点 × 0.5 + 踢脚 × 0.1 |
| 葫芦 | **98** + 三条牌点 × 0.2 + 一对牌点 × 0.1 |
| 同花 | **78** + 五张牌点之和 × 0.05 |
| 顺子 | **62** + 最大牌点 × 0.1 |
| 三条 | **24** + 三条牌点 × 0.2 + 踢脚之和 × 0.05 |
| 两对 | **16** + 高对 × 0.1 + 低对 × 0.1 + 踢脚 × 0.05 |
| 一对 | **9** + 对子牌点 × 0.1 + 踢脚之和 × 0.05 |
| 高牌 | **4** + 最高牌点 × 0.1 + 次高第三高之和 × 0.05 |

代码：`score_v4` / `SCORING_V4`（Python 默认）；PoC 为 `scoreV4` / `scoreActive`（[`poc/src/engine/scoring.ts`](../poc/src/engine/scoring.ts)）。

### 4.1 v3 vs v4 对比（20,000 局）

| 指标 | v3 | v4 |
|------|-----|-----|
| 单局总分均值 | 127.9 | **116.9** |
| 标准差 | 18.7 | **24.0** |
| 95 分位 | 164.6 | **171.5** |
| 最高分 | 252.9 | **332.9** |
| 含 ≥1 顺子 | 161.7 | **167.7** |
| 含 ≥1 premium | 166.6 | **179.3** |

**解读：** v4 降低高牌/一对 baseline（约 −11 分局均），但强牌单线分值更高，premium 局天花板与尾部更宽——鼓励追顺子/同花而非仅靠多线一对堆分。

## 7. 原型使用

### CLI 可玩原型

```bash
cd prototype
python -m quintet.cli
```

支持：设置 k、选择 v1/v2/v3/v4 计分、`place <池索引> <行> <列>`、`auto` 随机合法着法。

### 程序化 API

```python
from quintet.game import SoloGame
from quintet.scoring import SCORING_V4

game = SoloGame(pool_size=2, scoring=SCORING_V4)
game.start()
# game.play(pool_index, row, col)
result = game.final_score()  # after 25 placements
```

## 8. 后续调优建议

1. **策略型模拟：** 贪心 AI（500 局，k=2）单局均值 **162.6** vs 随机 **123.6**，说明技巧提升空间充足
2. **k 值分档：** k=1 时可略降 baseline；k=4 竞技模式维持 v2
3. **双人模式：** 贪心 vs 随机胜率约 97%（k=2），共享牌池封锁效应明显；需人类对战 playtest
4. **同分规则：** 已实现于 `two_player.resolve_match` — 总分 → 单局最高 → 强牌线路数 → 平局

## 9. Phase 3 原型（Python CLI）

- **本地双人：** `cd prototype && python -m quintet.cli --mode versus`
- **双局制：** `cd prototype && python -m quintet.cli --mode match`
- **贪心 bot：** 对战模式中 `ai` 命令；`cd prototype && python -m quintet.simulate --greedy`
- **代码规范：** 代码、注释、日志均为英文
